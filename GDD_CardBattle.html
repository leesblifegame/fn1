2. 속성 시스템

5가지 속성이 공격·방어·스킬에 관여한다. 속성 간 상성은 추후 정의.

속성기호특성 (초안)
불 火Fire고화력, 낮은 방어
물 水Water지속 딜, 디버프
숲 木Forest회복, 버프 지원
빛 光Light실드, 정화
암 暗Dark흡수, 약화, 반전

3. 카드 시스템

3-1. 캐릭터 카드

수집 대상인 메인 카드. 각 캐릭터는 주속성 1개를 가지며 공격 3속성·방어 3속성을 보유한다.
공격 속성과 방어 속성은 주속성을 반드시 공유하여 카드가 낭비되지 않도록 설계.

캐릭터 구조 예시 — [불] 계열 캐릭터

속성 1 (주)속성 2속성 3
공격불물숲
방어불빛암

→ 불카드: 공격·방어 모두 사용 가능 (주속성, 100% 효율)
→ 물·숲카드: 공격만 사용 (비주력, 30~50% 효율)
→ 빛·암카드: 방어만 사용 (비주력, 30~50% 효율)

항목내용
기본 스탯HP, ATK, DEF, SPD (속성별 배율 적용)
고유 스킬1개 (항상 장착, 해당 캐릭터 전용)
장착 스킬 슬롯2개 (전투 전 셋팅, 범용 스킬 풀에서 선택)
커맨드 덱기본 40장 + 프리미엄 10장

3-2. 명령 카드 (커맨드 덱)

기본 덱 무료

전략적 차별화 포인트

4-2. 공격

동일 속성 카드를 1~5장 선택하여 공격 커맨드 실행.

사용 장수배율 (주속성)배율 (비주력)비고
1장100%30%기본
2장210%60%+10% 보너스
3장330%90%+30% 보너스
4장460%120%+60% 보너스
5장600%150%+100% 보너스 (MAX)
비주력 효율 30% 확정 (플레이테스트 후 조정). 속성 상성은 상세 밸런싱 단계에서 추가.

4-3. 방어

공격과 동일한 카드 장수 구조. 방어값이 피해를 상쇄한다.

자신의 턴에 선제 실드 선언 (확정). 방어 커맨드를 사용하면 해당 턴 내 또는 지정 턴 동안 실드가 피해를 우선 흡수한다.

4-4. 스킬

종류발동 조건비고
고유 스킬캐릭터 전용 조건항상 장착
장착 스킬 A/B동일속성 or 하이브리드전투 전 셋팅

스킬 예시 — 어둠의 치유

항목내용
발동 조건암 3장 + 빛 1장 (하이브리드)
카드 소모4장 소모
효과 1아군 1명에게 실드 생성 (1턴간 피해 우선 흡수)
효과 2해당 아군의 실드량을 3턴간 n씩 증가
분류방어 커맨드 스킬

손패에서 선택한 카드 목록의 속성 조합이 스킬의 requiredAttributes[]와 requiredCounts[]를 충족하면 발동 가능.
예) 암3 빛1 → requiredAttributes: [Dark, Light], requiredCounts: [3, 1]

ScriptableObject 설계

// ── 속성 열거 ──────────────────────────────────
public enum Attribute { Fire, Water, Forest, Light, Dark }

// ── 명령카드 ───────────────────────────────────
[CreateAssetMenu(menuName = "CardBattle/CommandCard")]
public class CommandCardData : ScriptableObject {
    public Attribute attribute;
    public bool isPremium;
}

// ── 스킬 트리거 조건 ───────────────────────────
[System.Serializable]
public class SkillTrigger {
    public Attribute[] attributes;   // ex) [Dark, Light]
    public int[]       counts;       // ex) [3, 1]
}

// ── 스킬 효과 ──────────────────────────────────
[System.Serializable]
public class SkillEffect {
    public SkillEffectType type;     // Shield, Heal, Damage, Buff ...
    public float value;
    public int   duration;           // 턴 수 (0 = 즉발)
    public bool  applyOverTime;      // true = 매턴 n씩 증가
}

// ── 스킬 데이터 ────────────────────────────────
[CreateAssetMenu(menuName = "CardBattle/Skill")]
public class SkillData : ScriptableObject {
    public string      skillName;
    public SkillTrigger trigger;
    public SkillEffect[] effects;    // 복수 효과 지원
}

// ── 캐릭터 카드 ────────────────────────────────
[CreateAssetMenu(menuName = "CardBattle/CharacterCard")]
public class CharacterCardData : ScriptableObject {
    public string      characterName;
    public Attribute   primaryAttribute;

    public Attribute[] attackAttributes;   // 3가지
    public Attribute[] defenseAttributes;  // 3가지

    public int   baseHP, baseATK, baseDEF, baseSPD;

    public SkillData   uniqueSkill;
    // 장착 슬롯은 런타임에 PlayerData가 보관
}

핵심 매니저 구조

GameManager          ← 씬/상태 총괄
  ├ TurnManager      ← 턴 순서, 드로우 타이밍
  ├ DeckManager      ← 셔플, 드로우, 버리기
  ├ HandManager      ← 손패 UI & 카드 선택
  ├ BattleUnit       ← HP/실드/버프 상태 관리
  ├ SkillResolver    ← 하이브리드 조합 판별 & 스킬 발동
  ├ DamageCalc       ← 속성 배율 적용 데미지 계산
  └ AIController     ← PvE AI (State Machine)

턴 흐름 구현 (State Machine)

public enum BattleState {
    Setup,          // 덱·스킬 셋팅
    DrawInitial,    // 5장 드로우
    PlayerTurn,     // 카드 선택 → 커맨드 실행
    EnemyTurn,      // AI 커맨드 실행
    DrawPhase,      // 3장 드로우
    CheckEnd,       // 승패 판정
    Result
}

속성 배율 계산 (DamageCalc)

public static float CalcMultiplier(Attribute used, CharacterCardData ch, bool isAttack) {
    Attribute[] validAttrs = isAttack ? ch.attackAttributes : ch.defenseAttributes;
    if (!validAttrs.Contains(used)) return 0f;       // 사용 불가 속성
    float baseMulti = (used == ch.primaryAttribute) ? 1.0f : 0.3f;
    return baseMulti;
}

public static float CardCountBonus(int count) {
    // 1장=×1.0, 2=×2.1, 3=×3.3, 4=×4.6, 5=×6.0
    float[] table = { 0, 1.0f, 2.1f, 3.3f, 4.6f, 6.0f };
    return table[Mathf.Clamp(count, 1, 5)];
}

하이브리드 스킬 판별 (SkillResolver)

public static bool CanActivate(SkillData skill, List<CommandCardData> selected) {
    var trigger = skill.trigger;
    for (int i = 0; i < trigger.attributes.Length; i++) {
        int have = selected.Count(c => c.attribute == trigger.attributes[i]);
        if (have < trigger.counts[i]) return false;
    }
    return true;
}

8. 개발 로드맵 (Phase 1 집중)

단계목표주요 작업
Phase 1-A데이터 레이어ScriptableObject 정의, 캐릭터 3종·스킬 5종 시범 데이터 입력
Phase 1-B핸드 시스템셔플·드로우·버리기, 손패 UI(카드 드래그)
Phase 1-C1v1 전투 코어TurnManager, DamageCalc, 공격/방어 커맨드
Phase 1-D스킬 시스템SkillResolver, 효과 적용(실드·버프·지속효과)
Phase 1-EAI 상대룰 기반 AI (주속성 우선 사용, 위기시 방어)
Phase 23v3 확장멀티 유닛 관리, 타겟 선택 UI
Phase 3콘텐츠캐릭터 확장, 덱 빌딩, 스테이지 PvE
Phase 45v5 / 12v12자동전투 모드, 포지션 시스템
확정된 규칙 요약
· 방어: 자신의 턴에 선제 실드 선언
· 덱 소진: 버린 카드 재순환, 부족한 장만큼 지급, 패널티 없음
· 속성 상성: 현재 없음, 추후 상세 밸런싱 때 추가
· 비주력 효율: 30% 고정 (플레이테스트 후 조정)
